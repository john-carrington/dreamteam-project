# Skill Map - Карты экспертизы и развития сотрудников
Внутренний сервис компании Naumen для поиска и структурирования экспертизы сотрудников в образовательных и HR-задачах.

## О проекте

HR и команды развития часто сталкиваются с одними и теми же вопросами:

-   **Кого отправить спикером на конференцию?**
-   **Кто реально разбирается в узкой теме** (например, usability testing)?
-   **Кого пригласить в жюри** на студенческий конкурс?
-   **Кто может стать ментором** или внутренним преподавателем?

Экспертиза сотрудников есть, но она не структурирована, не подтверждена и не используется для развития и внешнего позиционирования компании.

**Карта экспертизы** — это сервис, который делает экспертизу видимой, достоверной и доступной за несколько кликов.

----------

## Функционал

### Профиль экспертизы сотрудника

Каждый сотрудник может создать и поддерживать свой профиль:

-   **Навыки** — профессиональные (Java, QA, UX Research) и экспертные (публичные выступления, менторство)
-   **Уровень владения** — базовый / уверенный / эксперт
-   **Опыт** — участие в проектах, выступления, обучение и менторство
-   **Статус готовности** — готов выступать / быть ментором / участвовать в жюри

Поддерживаемые операции: создание и редактирование профиля, добавление/удаление навыков, обновление статуса готовности.

----------

### Система подтверждения навыков

Для формирования доверия к указанной экспертизе:

-   Один сотрудник может подтвердить навык другого
-   Подтверждение привязывается к конкретному навыку
-   Отображается количество подтверждений и список подтвердивших
-   Встроенная защита от накрутки: один пользователь — одно подтверждение на навык

----------

### Поиск экспертов

HR-специалист может быстро найти нужного кандидата:

-   Поиск по запросу (например: `"Java + жюри"` или `"usability testing спикер"`)
-   **Фильтры**: навыки, уровень, тип активности (спикер / ментор / жюри), готовность участвовать
-   **Результат**: список кандидатов с краткой информацией — навыки, уровень, подтверждения, релевантный опыт
-   Сортировка по количеству подтверждений или релевантности

----------

### Подбор и приглашение кандидатов

Полный сценарий работы HR:

1.  Формирование запроса
2.  Получение списка подходящих кандидатов
3.  Выбор одного или нескольких
4.  Отправка приглашения
5.  Отслеживание статуса отклика (принял / отказался)

## Запуск приложения

### Требования

-   Docker

### Установка и запуск

```bash
# 1. Клонируйте репозиторий
git clone https://github.com/john-carrington/dreamteam-project.git
cd dreamteam-project

# 2. Настройте переменные окружения
cp .env.example .env
# Откройте .env и заполните необходимые значения

# 3. Собрать и запустить контейнеры
docker compose up -d --build

```

### Доступ к приложению

После запуска откройте в браузере: [http://localhost](http://localhost/)
